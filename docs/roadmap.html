Dashboard de administración
Julio – Agosto 2026
En progreso

🎯 Principio rector: Asistir, no automatizar

  • El sistema prepara el post (texto optimizado por canal + imagen adjunta). Roberta lo aprueba y publica ella. Cero APIs de Meta — cero dependencias que se rompan.
  • Botón "Copiar para Facebook" / "Copiar para Instagram" → pega y publica manualmente. Admin puede editar el texto generado antes de copiar.
  • Admin marca "Publicado" usando los campos posted_facebook, posted_instagram, posted_website que ya existen en DB → sistema registra automáticamente con published_at.
  • KPI de 48h calculable desde created_at vs published_at. Dato para el board.
  • Diseño móvil primero — Roberta y Kelsey operan desde el celular.
  • Criterio de sostenibilidad: cada pieza debe ser operable cuando Francisco no esté. Si solo él sabe mantenerla, no cuenta como done.
